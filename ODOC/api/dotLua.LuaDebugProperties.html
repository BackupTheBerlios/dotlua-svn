LuaDebug Properties

The properties of the LuaDebug class are listed below. For a complete list of LuaDebug class members, see the LuaDebug Members topic.

Public Instance Properties

Caller Returns the parent level (the caller) for the given debug level. If it has been constructed from a given level and the level has a parent it returns a LuaDebug object representing the caller of this level, otherwise a LuaException is being thrown.
CurrentLine Retrieves the current line of execution.
HasCaller Safe check if the current level has a parent level (a caller).
Level Retrieves the level from which the debug object has been created.
Line Retrieves the line number where the definition of the function starts. If no information is available it retrieves -1.
Name Retrieves a reasonable name for the given function. Because functions in Lua are first class values, they do not have a fixed name: Some functions may be the value of multiple global variables, while others may be stored only in a table field. The lua_getinfo function checks how the function was called or whether it is the value of a global variable to find a suitable name. If it cannot find a name, then name is set to null.
NameWhat Returns an explanation for the "Name" property. The value of namewhat can be "global", "local", "method", "field", or "" (the empty string), according to how the function was called. (Lua uses the empty string when no other option seems to apply.)
ShortSource A shorter "printable" version of Source.
Source Returns the name of the source. If the function was defined in a string the name of the string is being returned. If the function was defined in a file, then source starts with a '@' followed by the file name.
UpValues Retrieves the number of upvalues of the function.
What Returns the string "Lua" if this is a Lua function, "C" if this is a callback function, "main" if this is the main part of a chunk, and "tail" if this was a function that did a tail call. In the latter case, Lua has no other information about this function.
