What is LUA?

LUA is a leightweight, fast, structual scripting language, which is fully garbage collected. For more informations see the LUA project page.

What is the dotLua project?

dotLua is a C# class library which allows C# 2.0 to interact with the LUA scripting engine. It bases on the original, but slightly modified, LUA C API.

What does "slightly modified" mean?

Short answer: dotLua is not compatible with the existing Win32 builds linked on the LUA project site. dotLua comes with a Windows Dynamic Link Library (DLL) "lua_stdcall.dll", on which it depends on. Neither the LUA language nor it's standard libraries where altered, but we altered the C API so it can be used with C#.

Long technical answer: dotLua uses interop services to reach the C API from LUA. In the default built all LUA C API functions and function pointers are compiled by using the C calling convention. The API functions itself aren't a problem for C# but the callback function pointers are. There is no direct way to marshal a delegate to a function pointer with the C calling convention (cdecl) in C#. Therefore we compiled our own binaries using the standard calling convention (stdcall), and additionaly all C structures were aligned to 1 for easier interopability.

So the callback mechanism works?

Yes it does. It is implemented by using delegates. See the online documentation for further details.

Does dotLua come with its own library engine?

Yes. dotLua can load classes, either directly, from an assembly, or from a file being an assembly, and automatically create an interface for LUA. See the online documentation for further informations.

Does dotLua come with its own LUA library, like LUA Cheia?

No. Although this sub project is on the todo list.
